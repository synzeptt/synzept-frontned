# Product Principles

Synzept exists to help users think clearly, stay organized, maintain continuity, reduce mental overload, continue work naturally, remember important context, and execute consistently.

## Core Pillars

1. Memory: remember important user context.
2. Continuity: let users continue without repeating themselves.
3. Organization: reduce chaos and cognitive overload.
4. Intelligence: provide useful reasoning and structured assistance.
5. Calm experience: remain smooth, focused, and dependable.

## Feature Filter

Before implementation, every feature must answer:

- Does this improve continuity?
- Does this improve organization?
- Does this improve memory usefulness?
- Does this improve clarity or focus?
- Does this improve daily usefulness?
- Does this preserve calm UX?
- Does this avoid unnecessary complexity?

If the answer is mostly no, do not implement it for V1.

## V1 Priorities

- memory improvements
- retrieval quality
- project continuity
- better summaries
- better organization
- smoother chat
- task clarity
- daily assistance
- faster context restoration

Detailed launch scope is defined in `v1_completion_boundary.md`. Work outside that boundary should be deferred unless it is required to stabilize a V1 system.

## V1 Non-Priorities

- autonomous agents
- browser automation
- social features
- avatars
- gamification
- complex workflow builders
- excessive integrations
- entertainment features

## AI Behavior

The AI should be calm, strategic, useful, and consistent. It should avoid exaggerated personality, manipulation, excessive excitement, and over-talking.

## Retention Philosophy

Users should return because Synzept reduces cognitive load and preserves useful context, not because of attention mechanics.
