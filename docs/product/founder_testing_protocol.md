# Synzept Founder Testing And Product Evaluation Protocol

Synzept should improve through real workflow evaluation, observed friction, retention behavior, continuity failures, UX testing, and founder review.

The goal is to improve retention, continuity, usefulness, trust, and calm daily usage. The goal is not feature accumulation or hype-driven development.

## Core Evaluation Question

Every major product decision should answer:

Does this improve the core Synzept experience of memory, continuity, organization, intelligence, and calm UX?

If the answer is unclear, the feature or change should be deprioritized until a real workflow shows the need.

## Mandatory Evaluation Categories

Evaluate every major feature or refinement across:

- Usefulness
- Continuity quality
- Memory relevance
- UX smoothness
- Response quality
- Performance
- Trustworthiness
- Organizational clarity

## Founder Workflow Tests

The founder should regularly test Synzept by simulating real usage, not demo paths.

Use these scenarios:

- Continue a previous project after a gap.
- Restore interrupted work from dashboard or chat.
- Manage tasks created from real priorities.
- Organize messy notes into a project.
- Brainstorm with existing project context.
- Plan the next day from current goals.
- Ask Synzept to retrieve a past decision or priority.
- Use the dashboard as the first screen of the day.

During each test, actively look for friction, confusion, retrieval failures, continuity breaks, UX overload, slow responses, and generic AI behavior.

## Memory Quality Tests

For memory-driven workflows, answer:

- Did Synzept retrieve the right context?
- Was the retrieved context actually useful?
- Did the AI feel aware of ongoing work?
- Did memory reduce cognitive effort?
- Was there memory noise or irrelevant injection?
- Was context repeated unnecessarily?
- Would the user trust this memory behavior tomorrow?

Document any memory that is wrong, stale, noisy, missing, or over-weighted.

## Continuity Tests

For continuity workflows, answer:

- Can the user instantly continue where they left off?
- Does the dashboard surface unfinished work clearly?
- Does project context restore the right notes, tasks, memories, and conversations?
- Does chat continuation avoid making the user re-explain?
- Does Synzept preserve momentum without adding clutter?

The desired user reaction is: "I can instantly continue where I left off."

## AI Response Quality Tests

Evaluate whether responses are:

- Useful
- Structured
- Clear
- Concise
- Context-aware
- Calm in tone
- Specific to the user's actual workspace

Flag responses that are generic, repetitive, overlong, emotionally fake, vague, or disconnected from memory/project context.

## UX Tests

Evaluate:

- Navigation clarity
- Loading smoothness
- Dashboard readability
- Streaming quality
- Project organization
- Task simplicity
- Settings and memory control clarity
- Error recovery guidance

The interface should feel calm, intentional, lightweight, and dependable.

## Performance Tests

Track:

- Dashboard load speed
- Streaming startup time
- Retrieval latency
- Orchestration latency
- Frontend responsiveness

Performance is part of the product experience. Slow continuity feels like broken continuity.

## Retention Review

After each testing cycle, answer:

- Would a user naturally return tomorrow?
- Did Synzept reduce mental overhead?
- Did Synzept help the user organize work or life context?
- Did continuity feel valuable?
- Did memory feel trustworthy?
- Did the dashboard create a useful starting point?

These are the retention drivers that matter for V1.

## Refinement Rule

For each discovered issue:

1. Identify friction.
2. Isolate root cause.
3. Implement the smallest focused improvement.
4. Test thoroughly.
5. Measure impact.
6. Preserve architectural consistency.

Avoid large rewrites, duplicated systems, and feature expansion unless real workflows prove the need.

## Product Principle

Users should trust Synzept because it consistently remembers context, helps maintain momentum, reduces cognitive load, and organizes life and work smoothly.

Synzept should not rely on gimmicks, novelty, or hype.
